### 목표

- **입력(Input) → 처리(Logic) → 출력(Output)** 흐름을 몸으로 익힌다.
- 터미널에서 `python 파일.py`로 실행하는 습관을 만든다.
- “작게 만들고 → 실행해보고 → 오류를 고치고 → 개선한다” 과정을 경험한다.

---

### 개발 흐름이란 무엇인가?

> **개발 흐름이란**  
> 사용자의 입력이 들어와서  
> → 프로그램이 판단하고 처리한 뒤  
> → 결과를 다시 사용자에게 돌려주는  
> **전체 사고의 흐름**을 말한다.

이를 가장 단순한 형태로 표현하면 항상 이 3단계다.
```
입력(Input) → 처리(Logic) → 출력(Output)
```

### 오늘 만든 콘솔 프로그램의 공통 구조
|파일|입력|처리|출력|
|---|---|---|---|
|`hello.py`|❌|❌|print|
|`greet.py`|input(name)|문자열 처리|print|
|`calc_sum.py`|숫자 입력|int 변환 + 계산|결과 출력|
|`age_check.py`|나이 입력|조건 판단(if)|분기 출력|
|`repeat_menu.py`|메뉴 선택|조건/반복|화면 출력|
|`mini_budget.py`|금액/선택|누적/차감|잔액 출력|

### 5) 반복문으로 “여러 번 실행” 만들기 (Loop)

새 파일: `repeat_menu.py`
```python
while True:
    print("\\n=== 메뉴 ===")
    print("1) 인사하기")
    print("2) 두 수 더하기")
    print("0) 종료")

    choice = input("선택: ")

    if choice == "1":
        name = input("이름: ")
        print(f"{name}님 안녕하세요!")

    elif choice == "2":
        a = int(input("a: "))
        b = int(input("b: "))
        print("결과:", a + b)

    elif choice == "0":
        print("프로그램을 종료합니다.")
        break

    else:
        print("잘못된 입력입니다. 0~2 중에서 선택하세요.")
```

실행 예시
```
=== 메뉴 ===
1) 인사하기
2) 두 수 더하기
3) 종료
선택: 1
이름: 민수
민수님 안녕하세요!

=== 메뉴 ===
1) 인사하기
2) 두 수 더하기
3) 종료
선택: 2
a: 3
b: 4
결과: 7

=== 메뉴 ===
1) 인사하기
2) 두 수 더하기
3) 종료
선택: 0
프로그램을 종료합니다.
```

✅ 핵심:

- `while True` + `break`는 “프로그램 계속 실행” 구조
- 실제 서비스는 서버가 계속 실행되는 것과 비슷한 감각을 준다.

### 6) 오류를 일부러 내보고 “디버깅 감각” 익히기 (중요)

실습: 아래 코드로 실행해보기 (`bug_demo.py`)
```python
a = input("숫자 입력: ")
print(a + 10)  # 일부러 오류!
```

실행 결과(예시)

```
숫자 입력: 3
Traceback (most recent call last):
  File "bug_demo.py", line 2, in <module>
    print(a + 10)
TypeError: can only concatenate str (not "int") to str
```

✅ 해석:

- `a`는 문자열(str)
- `10`은 정수(int)
- 문자열 + 정수는 불가능 → TypeError

수정 코드
```python
a = int(input("숫자 입력: "))
print(a + 10)
```

### 7) 미니 프로젝트: “간단한 가계부”

목표

- 메뉴 기반 콘솔 앱 만들기
- 입력/처리/출력/반복/조건문 모두 사용

파일: `mini_budget.py`
```python
total = 0

while True:
    print("\\n==== 미니 가계부 ====")
    print("1) 수입 추가")
    print("2) 지출 추가")
    print("3) 현재 잔액 보기")
    print("0) 종료")

    choice = input("선택: ")

    if choice == "1":
        money = int(input("수입 금액: "))
        total += money
        print(f"✅ 수입 {money}원 추가. 현재 잔액: {total}원")

    elif choice == "2":
        money = int(input("지출 금액: "))
        total -= money
        print(f"✅ 지출 {money}원 추가. 현재 잔액: {total}원")

    elif choice == "3":
        print(f"📌 현재 잔액은 {total}원 입니다.")

    elif choice == "0":
        print("가계부를 종료합니다.")
        break

    else:
        print("⚠️ 잘못된 입력입니다. 0~3 중에서 선택하세요.")
```


코드 전체 해석:
```
[사용자]
  ↓ (키보드 입력)
input()
  ↓
[프로그램]
  - 변수에 저장
  - 조건 판단
  - 계산
  - 상태 유지(total)
  ↓
print()
  ↓
[사용자 화면에 결과 표시]
```

이 흐름이 while문으로 반복된다
```
입력 → 처리 → 출력
 ↑____________↓
```
👉 이게 바로 “프로그램이 살아있다”는 감각

#### 개발자는 코드를 이렇게 “순서도로” 생각합니다.

(1) 미니 가계부의 흐름을 말로 풀면:

1. 프로그램 시작
2. 잔액을 0으로 초기화
3. 메뉴를 보여준다
4. 사용자의 선택을 입력받는다
5. 선택에 따라:
    - 수입이면 더한다
    - 지출이면 뺀다
    - 조회면 보여준다
    - 종료면 프로그램 종료
6. 다시 메뉴로 돌아간다
