학습 단계

### ① 기초 문법·구조 설계력 학습 (1~3일차)

- **1일차 – 개발 환경 구축 & 기본 문법**
    - WSL, VSCode 설치 → _개발자가 쓰는 실제 환경_을 그대로 사용
    - 변수, 자료형, 문자열 등을 배우며 **데이터를 다루는 최소 단위**를 이해
        
- **2일차 – 조건문·반복문을 이용한 로직 설계**
    - “조건에 따라 분기하기”, “여러 데이터를 반복 처리하기”를 배우며  
        → 앞으로 만들 모든 기능의 뼈대가 되는 **로직 설계 능력**을 기릅니다.
        
- **3일차 – 함수화 & 객체지향 구조화**
    - 반복되는 코드를 함수로 묶고, 클래스로 **데이터+동작을 하나의 설계도로 표현**하는 연습
    - 유지보수하기 좋은 구조를 처음부터 경험하게 됩니다.
        

### ② 확장성/협업 능력 학습 (4~7일차)

- **4일차 – 파일/모듈 나누기 & 표준 라이브러리 활용**
    - 하나의 긴 파일이 아니라, 실무에서처럼 **기능별로 파일과 모듈을 나누는 연습**
    - `os`, `pathlib`, `datetime` 등 표준 라이브러리를 사용해  
        “바퀴를 처음부터 다시 만들지 않는 법”을 배웁니다.
        
- **5일차 – TDD 기반 안정적 코드 작성 습관**
    - 테스트 코드부터 작성하고, 그 테스트를 통과하는 기능을 구현해보며  
        **“고치기 두려운 코드”가 아닌 “안심하고 수정 가능한 코드”**를 경험합니다.
        
- **6일차 – 외부데이터 수집**
    - `requests`, `BeautifulSoup` 등으로 외부 웹 페이지·API에서 데이터를 가져와  
        “내 코드가 혼자 노는 것이 아니라 **세상과 연결되는 경험**”을 합니다.
        
- **7일차 – GitHub 협업 & 버전 관리**
    - `clone / pull / fetch`, 브랜치 전략, PR, 코드 리뷰, 충돌 해결 등을 실습하면서  
        **팀 개발에 필수적인 협업 방식**을 몸으로 익힙니다.
        

### ③ UI & 데이터 활용을 통한 프로젝트 실행능력 완성 (8~10일차)

- **8일차 – UI 대시보드 구현(Streamlit)**
    - HTML/CSS를 몰라도, Streamlit으로 **사용자 인터페이스**를 만들어 보고  
        “코드를 눈으로 볼 수 있는 화면”을 직접 만들어 봅니다.
        
- **9일차 – 외부데이터 + API 연동 대시보드**
    - 파일/간단한 API 데이터를 불러와 화면에 표시하고  
        간단한 **대시보드 형태**로 정보를 정리합니다.
        
- **10일차 – 실전 미니 프로젝트 제작**
    - 1~9일차에 배운 내용을 모두 엮어서  
        **“나만의 작은 서비스”를 처음부터 끝까지 만들어 보는 날**입니다.
    - 이후 3일간의 가계부 프로젝트를 위한 **사전 리허설** 역할도 합니다.

실제 개발 워크플로우 : 회사에서 일어나는 일과 어떻게 연결되는지

- **개발 환경 세팅**
    - 회사 규칙에 맞는 Python 버전, 패키지, IDE, 컨테이너 환경 등을 준비
    - → 1일차 학습과 정확히 매칭
        
- **도메인 정의 & 요구사항 분석**
    - “무엇을 해결하는 서비스인가?”, “어떤 데이터를 다뤄야 하는가?”를 정의
    - → 2일차의 “조건/반복 로직 설계”, 3일차의 “데이터 구조 설계”와 연결
        
- **데이터 구조 & 핵심 기능 설계**
    - 어떤 클래스를 만들지, 어떤 함수가 필요할지, 어떤 책임을 나눌지 설계
    - → 3일차 “함수 & 객체지향 구조화”에서 연습
        
- **프로젝트 구조화(모듈 분리)**
    - `models.py`, `services.py`, `views.py` 처럼 역할별로 파일과 모듈 나누기
    - → 4일차 “파일/모듈 나누기 & 라이브러리 활용”
        
- **안정성 확보(TDD/테스트)**
    - 기능마다 테스트를 만들어두고, 배포 전/후에 계속 돌려보며 버그를 방지
    - → 5일차 “TDD/단위 테스트”와 동일한 사고방식
        
- **외부 데이터 연동(API, DB 등)**
    - 결제 API, 외부 지도 데이터, 외부 인증 서버 등과 연동
    - → 6일차 “웹 요청 & 크롤링”, 9일차 “외부데이터 & API 연동”
        
- **버전 관리 & 협업(Git)**
    - 여러 명의 개발자가 동시에 작업하고, 코드를 합치고, 충돌을 해결
    - → 7일차 “GitHub 협업 & 브랜치 전략”
        
- **UI 구현**
    - 프론트엔드/대시보드 화면을 만들어 사용자가 쓸 수 있는 형태로 제공
    - → 8일차 “Streamlit UI 구현”, 9일차 “대시보드화”
        
- **서비스 완성 & 배포**
    - 실행 가능한 상태로 묶고, 내부/외부 사용자에게 공유
    - → 10일차 “실전 미니 프로젝트 제작” + 결과물 공유 단계
        
- **운영 & 개선**
    - 실제 사용을 보면서 기능을 개선하고, 버그를 수정
    - → 11~13일차의 “가계부 프로젝트”에서 처음으로 맛보는 단계
